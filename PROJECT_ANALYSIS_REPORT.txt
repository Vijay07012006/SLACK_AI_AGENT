[High-Level Data Flow]
1. Slack Event occurs (team_join / member_joined_channel)
2. Bolt SDK receives event -> calls event handler.
3. Fetch User Info (Slack Web API) / Channel Info.
4. Research Phase (doBasicResearch):
   - If email is NOT personal -> scrape company website (getCompanyInfo).
   - Search GitHub by user name (getGithubInfo).
5. AI Analysis (analyzeWithAI):
   - LangChain prompt with Member Info + Research Data.
   - OpenAI returns JSON { fitScore, insights, recommendations }.
6. Database Save (saveMemberAnalysis) -> PostgreSQL 'member_analyses' table.
7. Post formatted Slack message to SLACK_PRIVATE_CHANNEL_ID.
8. Mark as sent (markAsSentToSlack).

3. 🐛 IDENTIFIED BUGS, MISTAKES & CRITICAL ISSUES
=========================================

[CRITICAL] Bug #1: Missing Environment Variable Validation
-----------------------------------------------------------
Lines: index.js (LINE ~30-40)
Issue: The prompt template uses 'process.env.COMPANY_NAME' and 'process.env.COMPANY_PRODUCT',
        but these are NOT in the 'requiredEnvVars' array.
Impact: If not set, the prompt will literally say "Our Company" / "Our Product".
        This reduces the quality of AI analysis significantly.
Fix:   Add them to requiredEnvVars OR set sensible defaults and log a warning.

[CRITICAL] Bug #2: Fragile JSON Parsing from OpenAI
---------------------------------------------------
Lines: index.js (analyzeWithAI function)
Issue: The regex to clean markdown is too weak.
        Code: .replace(/```json\n?|\n?```/g, "")
        Fails on:
          - "```json\n" (has trailing newline? handled)
          - "```JSON" (uppercase - fails)
          - "``` json" (space - fails)
          - Trailing commas in JSON (causes JSON.parse to throw)
Impact: If OpenAI returns anything unexpected, it falls back to default 50 score
        silently. Users won't know the analysis failed.
Fix:   Use a more robust regex: /```(?:json)?\s*([\s\S]*?)\s*```/i and extract group 1.
        Additionally, use JSON5 library or sanitize trailing commas.

[CRITICAL] Bug #3: Web Scraping is Highly Unreliable (Anti-Bot & Rate Limits)
-----------------------------------------------------------------------------
Lines: index.js (getCompanyInfo function)
Issue: Direct axios GET requests to unknown domains. Many modern websites:
       - Block requests without a proper User-Agent (you have one, good).
       - Have Cloudflare/DDOS protection (returns 403/503).
       - Return massive HTML payloads (parsing only title is okay, but pointless).
       - Might trigger rate-limiting on YOUR IP if multiple members join.
Impact: Wastes time (5 second timeout per URL), generates errors, and provides
        very little useful data ("Fetched from https://..." is useless for AI).
Fix:   Use a proper scraping service (Apify, BrightData) OR use Clearbit/Hunter.io API
        for company enrichment instead of raw HTML scraping.

[HIGH] Bug #4: Channel Type Filter is Incomplete
------------------------------------------------
Lines: index.js (member_joined_channel event handler)
Issue: Code checks `if (event.channel_type === "C")`. Slack channel types:
       - "C" = Public channel
       - "G" = Private channel
       - "M" = Multiparty direct message
Impact: The bot will IGNORE members joining Private channels or Group DMs.
        The comment says "NOW FIXED" but it's only half-fixed.
Fix:   Remove the filter entirely OR expand it to ["C", "G", "M"] if you want all.

[HIGH] Bug #5: Database SSL Configuration May Fail in Production
----------------------------------------------------------------
Lines: db.js (Pool config)
Issue: In production (NODE_ENV !== 'development'), SSL is `{ rejectUnauthorized: true }`.
        Many cloud-hosted Postgres (Neon, Supabase, AWS RDS) require `rejectUnauthorized: false`
        OR a specific CA certificate. If set to 'true' with a self-signed or default cert,
        the connection will crash.
Impact: App crashes on startup in production.
Fix:   Set `rejectUnauthorized: false` universally for simplicity, OR add an env var
        'PG_SSL_CA' to load the certificate.

[MEDIUM] Bug #6: Redundant API Calls on Channel Join
-----------------------------------------------------
Lines: index.js (analyzeChannelJoin)
Issue: Every time a member joins ANY channel, it posts a notification to the
        private channel. In large workspaces, 50 people joining a general channel
        will flood the admin channel with 50 spam messages.
Impact: Channel spamming, noise.
Fix:   Only notify if channel is specific (e.g., "#introductions") OR implement
        a debouncing mechanism (e.g., send a daily digest).

[MEDIUM] Bug #7: Hardcoded AI Model Fallback
---------------------------------------------
Lines: index.js (constructor)
Issue: Model is set to "gpt-3.5-turbo". If OpenAI changes this model name or
        deprecates it, the app breaks.
Fix:   Move to env var: OPENAI_MODEL="gpt-4o-mini".

[MEDIUM] Bug #8: No Deduplication / Rate Limiting on team_join
---------------------------------------------------------------
Issue: Slack's `team_join` event can trigger multiple times for the same user
        (e.g., profile updates sometimes trigger it). Although DB has upsert,
        it will re-run the expensive AI analysis every time.
Impact: Wastes OpenAI credits and time.
Fix:   Check the 'updated_at' or 'analyzed_at' in DB. If analyzed within the last 24h, skip.

[MEDIUM] Bug #9: Express Error Handler Missing Async Support
------------------------------------------------------------
Lines: index.js (setupExpress)
Issue: The global error handler does NOT catch errors from async route handlers
        (e.g., if /test/analyze-member throws an async error, it won't trigger this).
Impact: Unhandled promise rejections in routes.
Fix:   Wrap route handlers with a try-catch OR use express-async-errors package.

[LOW] Bug #10: testdb.js uses Top-Level Await but No Error Handling
-------------------------------------------------------------------
File: testdb.js
Issue: Uses `await` directly. While valid in ESM, if the connection fails, it crashes
        without a proper catch block.
Impact: Cleanup (`pool.end()`) won't run if `await pool.query` throws.
Fix:   Wrap in try/catch.

[LOW] Bug #11: Slack WebClient Concurrency Limits Ignored
---------------------------------------------------------
Issue: If 10 members join simultaneously, `analyzeAndPostMember` runs 10 instances
        concurrently. Slack Web API has rate limits (50+ per minute).
Impact: May hit Slack rate limits causing temporary failures.
Fix:   Implement a queue (BullMQ or p-queue) to process events sequentially.

[LOW] Bug #12: Research Data Not Stored Properly (Empty Content)
----------------------------------------------------------------
Lines: index.js (getCompanyInfo)
Issue: `content: `Fetched from ${url}`` is useless. The AI prompt gets generic text.
        It doesn't extract the actual description/paragraphs from the page.
Impact: AI has very little context about the company.
Fix:   Use Cheerio to extract <meta name="description"> or first paragraph.

[LOW] Bug #13: Missing Slack Permission Scopes in Documentation
----------------------------------------------------------------
Issue: The bot needs specific OAuth scopes (users:read, channels:read, chat:write,
        groups:read). Not documented anywhere.
Impact: New developers setting up the project will struggle with permissions.
Fix:   Add a .env.example file with a list of required scopes.

[LOW] Bug #14: No Logging Rotation
----------------------------------
Issue: Console logs are great, but in production they go to stdout. No file-based
        logging or log rotation.
Impact: Hard to debug historical issues.
Fix:   Add Winston or Pino with file transports.
